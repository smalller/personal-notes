ajax：
	本质：使用js提供的异步对象 XMLHttpRequest 异步的向服务器提交请求，并且接收服务器响应回来的数据，同一时间，能同时处理多个任务
	步骤：
		一、创建异步对象
			var xhr = new XMLHttpRequest();
			
		二、创建请求
			xhr.open("请求方式","请求地址url",是否使用异步);
			如：xhr.open("get","http://127.0.0.1",true);
			
		三、发送请求
			xhr.send(formData);
			注意：在使用get请求方式时，没有formData请求主体，所以这里的参数可以不填，或者写null
		
		四、绑定监听，接收响应数据
		（1）xhr.readyState 属性：用于表示xhr对象的请求状态，值表示5个状态
				0：请求尚未初始化
				1：打开服务器连接，正在发送请求
				2：接收响应头
				3：接收响应主体
				4：响应接收完毕
			
		（2）绑定监听，接收响应
				xhr.onreadystatechange = function () {
				
					//该事件会调用4次，我们只关注最后一次，并且要求状态码为200（响应成功）
					if(xhr.readyState == 4 && xhr.status == 200) {
						
						//接收响应
						var result = xhr.responseText;
					}
			}
			
			

跨域：
	定义：一个网站下的网页，请求/使用了另一个网站下的资源
	问题：ajax不允许发送跨域请求
	原理：
		一、所有浏览器都有一个“同源策略”
		二、一个网站的网页中发送的ajax请求，只能访问自己网站提供的数据，不能访问别的网站提供的数据
		（1）允许发送请求
		（2）也能成功收到请求
		（3）浏览器会检查响应头中的来源地址，如果来源地址和当前网页所在网站的地址不同，则禁止使用该数据，就报错
	跨域请求包括：
	（1）域名不同
	（2）二级域名不同
	（3）端口不同
	（4）协议不同
	（5）相同端口和协议，IP和域名之间互访，也算跨域
		如：localhost:3000	和 127.0.0.1:3000	也算跨域
	解决：
	（1）CORS：只需要服务端在返回数据时，在响应头中伪造来源地址，和客户端浏览器地址中的地址保持一致
	（2）JSONP：
			前端：在每次要执行请求操作时，就咋body的结尾或head的开头动态创建<script src="服务器地址?callback=show"></script>标签，然后去请求后端，每请求完成一次，就会删除该条script标签
			后端：接收将要返回的数据和callback函数名，并将其动态拼接到要返回的函数调用语句中，再返回给前端的<script>标签
	


HTTP协议：
	一、通用头信息
		1.Request URL：请求的URL，对应浏览器地址栏；向服务器获取相关内容
		2.Request Method：请求的方法，获取内容的方法（get/post）
		3.Status Code：响应状态码
			1xx：正在请求，提示信息
			2xx：如200，表示响应成功
			3xx：如301，表示永久重定向；302表示临时重定向；304表示请求未改变，命中缓存
			4xx：如404，表示请求资源不存在；403，表示权限不够，405，表示请求方法不被允许
			5xx：如500，服务器代码出错
		3.Remote Address：请求的服务器IP地址以及端口号
		
	二、响应头信息
		1.Connection					连接方式，keep-alive 表示开启持久链接
		2.Content-Encoding		内容压缩格式，如：gzip
		3.Content-Type				响应的文件类型		
		4.Location							跳转的URL，常配合状态码3xx使用
		
	三、HTTP性能优化：
		1.减少连接创建的次数（开启持久连接）
		2.减少请求的次数（代码设计更合理）
		3.提高服务器端运行的速度
		4.尽可能减少响应数据的长度
